Comment créer un module PrestaShop personnalisé en 2025

Eddy RAKOTOARIVONY
15 Janvier 2025 8 min de lecture

Développer un module PrestaShop personnalisé est l'une des compétences les plus précieuses pour un développeur e-commerce. Que ce soit pour ajouter des fonctionnalités spécifiques à votre boutique ou pour créer une solution réutilisable, maîtriser le système de modules de PrestaShop vous donnera une flexibilité totale.

Dans ce guide complet, nous allons créer un module PrestaShop de A à Z, en suivant les bonnes pratiques de la version 8.

Prérequis

Avant de commencer, assurez-vous d'avoir :

  • Une installation PrestaShop 8.x fonctionnelle en local
  • PHP 8.1+ avec Composer installé
  • Un éditeur de code (VS Code recommandé)
  • Des connaissances de base en PHP orienté objet

Structure d'un module PrestaShop

Chaque module PrestaShop suit une structure de fichiers bien définie. Voici la structure minimale pour notre module que nous appellerons mymodule :

modules/
└── mymodule/
    ├── mymodule.php      ← Fichier principal obligatoire
    ├── logo.png          ← Logo 32x32px
    ├── config.xml        ← Cache de configuration
    └── views/
        └── templates/
            └── hook/
                └── display_home.tpl

Le fichier principal PHP

Le cœur de votre module est le fichier mymodule.php. Il doit contenir une classe qui hérite de Module :

<?php
if (!defined('_PS_VERSION_')) {
    exit;
}

class MyModule extends Module
{
    public function __construct()
    {
        $this->name    = 'mymodule';
        $this->tab     = 'front_office_features';
        $this->version = '1.0.0';
        $this->author  = 'Eddy RAKOTOARIVONY';
        $this->need_instance = 0;

        parent::__construct();

        $this->displayName = $this->l('Mon Module');
        $this->description = $this->l('Description de mon module.');
    }

    public function install()
    {
        return parent::install()
            && $this->registerHook('displayHome');
    }

    public function uninstall()
    {
        return parent::uninstall();
    }

    public function hookDisplayHome($params)
    {
        return $this->display(__FILE__, 'display_home.tpl');
    }
}

Les Hooks PrestaShop

Les hooks sont le mécanisme central du système de modules PrestaShop. Ils permettent d'injecter du contenu ou d'exécuter du code à des emplacements prédéfinis dans le cycle de vie de l'application.

Les hooks les plus utilisés sont :

  • displayHome — Affichage sur la page d'accueil
  • displayHeader — Dans le <head> de chaque page
  • actionCartSave — Lors de la sauvegarde du panier
  • displayProductAdditionalInfo — Informations produit supplémentaires

Créer le template Twig

Dans PrestaShop 8, les templates utilisent Twig. Créez le fichier views/templates/hook/display_home.tpl :

{extends file='customer/page.tpl'}

{block name='page_content'}
  <div class="my-module-block">
    <h2>{l s='Bienvenue' mod='mymodule'}</h2>
    <p>{l s='Mon module fonctionne !' mod='mymodule'}</p>
  </div>
{/block}

Installer votre module

Une fois vos fichiers créés, rendez-vous dans le back-office PrestaShop → Modules → Gestionnaire de modules. Votre module apparaîtra dans la liste. Cliquez sur "Installer" pour l'activer.

💡 Conseil : Activez le mode développeur de PrestaShop (Paramètres avancés → Performances) pour éviter les problèmes de cache lors du développement.

Bonnes pratiques

  • Toujours vérifier _PS_VERSION_ en début de fichier
  • Utiliser les méthodes de traduction $this->l()
  • Implémenter proprement les méthodes install() et uninstall()
  • Nettoyer la base de données lors de la désinstallation
  • Suivre les standards PSR-12 pour le code PHP

Conclusion

Vous avez maintenant les bases pour créer un module PrestaShop personnalisé. Le système de modules de PrestaShop est puissant et flexible — il vous permet d'étendre et de personnaliser votre boutique de manière infinie.
